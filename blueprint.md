## 2. 智能协作协议 (v3.0 Supervision Protocol)

### 高级功能逻辑新增
1. **动态任务划分：** 所有任务实时评估工作复杂度并动态分配至最佳模型。
2. **极限单元测试：** 强制覆盖关键路径 100%，同时增加代码覆盖率阈值警报 (>=85%)。
3. **任务上下文整合：** MCP (Model Context Protocol) 2.0 自动拼接上下游全链路数据，确保零上下文丢失。
4. **容错与救火：** 在 CI/CD 流程失败后触发附加的模型链路，提供冗余自愈机制。支持最多 5 次尝试。

> ✅ **模型搭配改进：** 平衡性能与成本，优先低延迟模型完成依赖链节点任务。

## 3. 验收标准与升级版开发流程

**3.1 验收标准（DoD）**
- **接口规范：** 所有 API 均采用 OpenAPI v3.1 规范。
- **性能目标：** 响应时间不得超过 150ms（高并发目标环境）。
- **协同性：** 自动生成关键模块文档并附带调用示例。
- **安全合规：** 代码静态分析不得存在高级别漏洞（参考 OWASP Top 10）。

**3.2 新开发 Workflow**
1. **PR 策略：** 通过语义 Git Commit 标准化提交日志，PR 需通过 Code Review 并成功触发 GitHub Actions 流水线。
2. **自动化部署：** 使用最新 Docker Container 工具链，支持选配 k8s 环境拼装。
3. **回归测试：** 合并前自动测试覆盖所有功能模块（重点覆盖边界用例）。

## 4. 操作指引：使用 Cursor v3.0.1+ 版本

结合本蓝图以及最新的工具链，甲方需遵循如下工作步骤：

### 第一步：准备工作环境
- **GitHub：** 创建并配置私有仓库，启用必要的 Webhooks。
- **云服务资源：** 申请并绑定 AWS IAM 角色，完成 S3 和 Lambda 的权限配置。
- **Cursor 工具：** 确保 Cursor 最新版本已安装，并正确配置所有模型访问密钥 (API Key)。

### 第二步：规则固化
- **协议绑定：** 粘贴最新“智能协作协议（v3.0）”至 Cursor 的 Settings > Rules 模块。
- **预检查：** 对接本地任务队列，确保模型调用链正常运行。

### 第三步：发起核心任务
- 在 Cursor Composer 中明确项目目标，例如：
  - “完成商品推荐系统服务端模块开发，并生成配套文档”。

### 第四步：审查与迭代
- 借助生成的 PR 自动完成代码审查、快速部署并生成演示链接。
- 在预览环境验收后，尝试一键迭代优化功能，直至最终通过。
